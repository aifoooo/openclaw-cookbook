# 龙虾初相遇

OpenClaw（龙虾）是一个会成长的 AI 助手。以下是我在使用过程中积累的经验，供参考。

---

## 一、搜索体验升级

龙虾内置 `web_search` 和 `web_fetch` 工具，具备联网能力。但默认搜索质量一般，结果不够精准。

**推荐方案**：安装 Tavily 技能。

```bash
# 跟龙虾说："需要安装 Tavily"
# 它会自动安装，最后问你要 API Key
```

**获取 API Key**：前往 [Tavily 官网](https://tavily.com) 注册，免费额度足够日常使用。

---

## 二、核心准则

龙虾的行为取决于你给它的"价值观"。以下准则让它更靠谱：

| 准则 | 说明 |
|------|------|
| 绝对不臆造 | 不确定就先核实，输出基于可验证事实 |
| 穷尽方法解决 | 不轻易说"做不到"，有限制就提供替代方案 |
| 能做就不推给用户 | 凡是能完成的任务，绝不推给用户 |
| 犯过错不再犯 | 从错误中学习，避免重复踩坑 |

**配置方式**：写入 `MEMORY.md`，让龙虾永久遵循。

---

## 三、会话重置问题

**现象**：第二天龙虾就忘记昨天的事。

**原因**：OpenClaw 默认每天凌晨 4 点重置会话。

**解决方案**：配置空闲重置，14 天不活跃才重置。

```json
"session": {
  "reset": {
    "mode": "idle",
    "idleMinutes": 20160
  }
}
```

**补充**：聊多了可能崩，崩后 `/new` 恢复但记录丢失。重要内容及时写入文件。

---

## 四、上下文溢出

**现象**：聊多了报错"上下文超出大模型限制"。

**解决方案**：配置压缩和剪枝策略。

```json
"compaction": {
  "mode": "safeguard",
  "reserveTokens": 60000,
  "keepRecentTokens": 80000,
  "memoryFlush": {
    "enabled": true,
    "softThresholdTokens": 15000
  }
},
"contextPruning": {
  "mode": "cache-ttl",
  "ttl": "1h",
  "keepLastAssistants": 3,
  "softTrimRatio": 0.3,
  "hardClearRatio": 0.5
}
```

---

## 五、模型降级

**场景**：主模型调用超限或欠费。

**解决方案**：配置降级链，自动切换备用模型。

```json
"model": {
  "primary": "tencentcodingplan/glm-5",
  "fallbacks": [
    "tencentcodingplan/hunyuan-2.0-thinking",
    "deepseek/deepseek-reasoner"
  ]
}
```

---

## 六、能力持久化

**现象**：龙虾忘记曾经安装的工具和能力。

**解决方案**：封装成 skill。

PDF 转换、深度研究、视频截图……都可以封装成 skill。一旦封装，永久拥有。

**skill 目录**：`~/.openclaw/workspace/skills/`

---

## 小结

以上配置大多可以通过对话完成——跟龙虾说需求，它会帮你搞定。

OpenClaw 的魅力在于：**它不是一成不变的工具，而是会成长的伙伴**。
