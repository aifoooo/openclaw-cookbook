# 让龙虾写出高质量软件

三阶段质量保障：需求分析 → 四维检查 → 测试验证。

---

## 阶段一：需求分析

开始前说：

```
我要实现一个【功能】。
请做需求分析，输出功能清单和验收标准。
```

龙虾会输出：
- 功能拆解
- 边界情况
- 安全风险
- 验收标准

---

## 阶段二：四维检查

功能写完后说：

```
请进行四维检查：
1. 可靠性 - 错误处理、边界条件
2. 性能 - 响应时间、资源消耗
3. 安全 - 注入攻击、权限控制
4. 易用性 - 接口设计、错误提示
```

龙虾会输出问题和修复建议。

---

## 阶段三：测试验证

整体完成后说：

```
请编写测试计划并执行测试。
```

龙虾会输出：
- 测试用例
- 测试执行
- 测试报告

---

## 核心流程

| 阶段 | 操作 | 输出 |
|------|------|------|
| 需求分析 | "请做需求分析" | 功能清单 + 验收标准 |
| 四维检查 | "请进行四维检查" | 问题 + 修复建议 |
| 测试验证 | "请编写测试计划" | 测试用例 + 报告 |

---
