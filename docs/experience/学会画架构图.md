# 让龙虾学会画架构图

OpenClaw 本身不会画架构图，但可以通过 **技能封装** 学会。

本文分享如何让龙虾从"一脸懵逼"变成"秒画架构图"。

---

## 问题背景

**场景**：你需要画一个架构图。

```
你：你现在有画架构图的技能吗
我：有的！我可以用 Mermaid 画架构图
```

**问题**：
- 只回答了"有"，没有深入
- 没有封装成 skill，每次都要临时写代码
- 下次对话又忘了

**理想状态**：
```
你：你现在有画架构图的技能吗
我：检查工具... 有 Graphviz 和 Mermaid，但没有封装成 skill。
    要不要我创建一个 diagram skill？
```

---

## 技能封装：diagram skill

### 核心思路

**skill = 工具 + 说明 + 触发词**

| 组件 | 作用 |
|------|------|
| **工具** | Graphviz（dot 命令）、Mermaid（mmdc 命令） |
| **说明** | SKILL.md 描述如何使用 |
| **触发词** | "画图"、"架构图"、"流程图" |

### 目录结构

```
skills/diagram/
├── SKILL.md        # 技能说明
├── generate.sh     # Graphviz 生成脚本
└── mermaid.sh      # Mermaid 生成脚本
```

### SKILL.md 核心内容

```markdown
---
name: diagram
description: "架构图生成工具。触发词：画图、架构图、流程图"
---

## 支持类型

| 类型 | 语法 | 适用场景 |
|------|------|---------|
| 架构图 | Graphviz DOT | 系统组件关系 |
| 流程图 | Graphviz DOT | 流程步骤 |
| 时序图 | Mermaid | 交互时序 |

## 使用方式

自然语言描述：
```
画一个架构图：
- Gateway 接收请求
- 转发到 Proxy
- Proxy 调用 LLM API
```
```

## 工具集成

### Graphviz

**安装检查**：
```bash
which dot && dot -V
# /usr/bin/dot
# dot - graphviz version 9.0.0
```

**生成图片**：
```bash
echo 'digraph G {
    A -> B;
    B -> C;
}' | dot -Tpng -o output.png
```

**优势**：
- 自动布局，不用调整位置
- 支持复杂图形结构
- 输出美观

### Mermaid

**安装检查**：
```bash
which mmdc && mmdc --version
```

**生成图片**：
```bash
echo 'graph TB
    A --> B
    B --> C' | mmdc -o output.png
```

**优势**：
- 语法简单，类 Markdown
- GitHub 原生支持
- 学习曲线平缓

## 自然语言理解

### 触发规则

**用户说**：
```
画一个 OpenClaw Monitor 的架构图
```

**我理解**：
1. 识别触发词"画"+"架构图"
2. 提取主体"OpenClaw Monitor"
3. 回忆已知信息（之前对话中提到的架构）
4. 生成 DOT 代码
5. 调用 Graphviz 生成 PNG

### 隐式指令识别

**关键改进**：

| 用户说法 | 隐式含义 |
|---------|---------|
| "你需要哪些skill才能..." | 检查能力 + 检查 skill + 缺就创建 |
| "有没有skill可以..." | 如果没有，建议创建 |
| "你会...吗" | 回答能力 + 是否需要 skill |

## 最佳实践

### 创建 skill 的步骤

**步骤一：确认工具可用**

```bash
# 检查 Graphviz
which dot

# 检查 Mermaid
which mmdc
```

**步骤二：创建 skill 目录**

```bash
mkdir -p ~/.openclaw/workspace/skills/diagram
```

**步骤三：编写 SKILL.md**

核心内容：
- `name`：技能名称
- `description`：描述 + 触发词
- 使用方式示例

**步骤四：测试**

```
画一个架构图：Gateway -> Proxy -> Backend
```

### 用户应该怎么说

**方式一：直接创建（最清晰）**
```
帮我创建一个画架构图的 skill
```

**方式二：描述需求**
```
我经常需要画架构图，有没有办法让我更方便地画图？
```

**方式三：隐式指令**
```
你会画架构图吗，或者你需要哪些skill才会画架构图？
```
