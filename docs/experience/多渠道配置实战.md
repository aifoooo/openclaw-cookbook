# 多渠道配置实战

当你需要同时服务多个场景（私人、工作、家人）时，单渠道就不够用了。本文分享多渠道配置的实战经验。

---

## 一、需求场景

我需要三个独立场景：

| 场景 | 渠道 | 用途 |
|------|------|------|
| 私人 | QQ（mime账号） | 日常对话、技术探索 |
| 家人 | QQ（wife账号） | 家人日常使用 |
| 工作 | 飞书 | 工作相关事务 |

**核心要求**：三个场景的记忆完全隔离，互不干扰。

---

## 二、多 Agent 架构

OpenClaw 的 Agent 是独立的工作空间，每个 Agent 有自己的：
- 记忆文件（MEMORY.md）
- 用户配置（USER.md）
- 会话历史
- 可选的独立模型

### 2.1 配置 Agent

在 `openclaw.json` 中配置：

```json
{
  "agents": {
    "defaults": {
      "model": {
        "primary": "tencentcodingplan/glm-5"
      },
      "workspace": "/root/.openclaw/workspace"
    },
    "list": [
      {
        "id": "main"
      },
      {
        "id": "mime-qq",
        "name": "mime-qq",
        "workspace": "/root/ws-mime-qq",
        "model": "tencentcodingplan/glm-5"
      },
      {
        "id": "wife-qq",
        "name": "wife-qq",
        "workspace": "/root/ws-wife-qq",
        "model": "tencentcodingplan/glm-5"
      },
      {
        "id": "mime-feishu",
        "name": "mime-feishu",
        "workspace": "/root/ws-mime-feishu",
        "model": "tencentcodingplan/glm-5"
      }
    ]
  }
}
```

**关键点**：
- `main` 是默认 Agent，没有指定 workspace
- 其他 Agent 都有独立的 workspace 目录
- 可以给不同 Agent 配不同模型

### 2.2 绑定规则

配置好 Agent 后，需要绑定渠道：

```json
{
  "bindings": [
    {
      "match": { "channel": "qqbot", "accountId": "mime" },
      "agentId": "mime-qq"
    },
    {
      "match": { "channel": "qqbot", "accountId": "wife" },
      "agentId": "wife-qq"
    },
    {
      "match": { "channel": "feishu", "accountId": "default" },
      "agentId": "mime-feishu"
    }
  ]
}
```

**绑定逻辑**：消息到达时，根据 `channel + accountId` 匹配对应的 Agent。

## 三、多账号配置

以 QQ 多账号为例：

```json
{
  "channels": {
    "qqbot": {
      "enabled": true,
      "accounts": {
        "mime": {
          "appId": "YOUR_QQ_APP_ID_1",
          "clientSecret": "your-secret",
          "allowFrom": ["*"]
        },
        "wife": {
          "appId": "YOUR_QQ_APP_ID_2",
          "clientSecret": "your-secret",
          "allowFrom": ["*"]
        }
      }
    }
  }
}
```

**踩坑提醒**：
- `allowFrom: ["*"]` 表示接受所有私聊
- 如果不配置 `defaultAccount`，会报警告，但不影响使用（因为绑定规则已明确路由）

## 四、独立记忆系统

每个 Agent 的记忆隔离还不够，我还需要长期记忆系统（memU）也隔离。

### 4.1 创建独立数据库

```bash
# 创建三个数据库
psql -U postgres -c "CREATE DATABASE memu;"
psql -U postgres -c "CREATE DATABASE memu_wife_qq;"
psql -U postgres -c "CREATE DATABASE memu_mime_feishu;"

# 启用 pgvector 扩展
psql -d memu -c "CREATE EXTENSION IF NOT EXISTS vector;"
psql -d memu_wife_qq -c "CREATE EXTENSION IF NOT EXISTS vector;"
psql -d memu_mime_feishu -c "CREATE EXTENSION IF NOT EXISTS vector;"
```

### 4.2 配置 memubot 服务

每个数据库对应一个 memubot 服务：

```bash
# /etc/systemd/system/memubot-mime-qq.service
[Unit]
Description=memUbot - mime-qq Memory Sync Daemon
After=network.target postgresql.service

[Service]
Type=simple
User=root
WorkingDirectory=/root/.openclaw/workspace/skills/memu
Environment="MEMU_DSN=postgresql://memu:YOUR_STRONG_PASSWORD@localhost:5432/memu"
Environment="MEMU_SESSIONS_DIR=/root/.openclaw/agents/mime-qq/sessions"
Environment="MEMU_DATA_DIR=/root/ws-mime-qq/memu-data"
ExecStart=/root/miniconda3/envs/memu/bin/python memubot_daemon.py
Restart=always

[Install]
WantedBy=multi-user.target
```

创建三个服务：
- `memubot-mime-qq.service` → memu 数据库
- `memubot-wife-qq.service` → memu_wife_qq 数据库
- `memubot-mime-feishu.service` → memu_mime_feishu 数据库

启动服务：

```bash
systemctl enable memubot-mime-qq memubot-wife-qq memubot-mime-feishu
systemctl start memubot-mime-qq memubot-wife-qq memubot-mime-feishu
```

## 五、Skills 统一管理

多个 Agent 共享同一套 Skills，避免重复维护。

### 5.1 统一存放位置

```
/root/.openclaw/workspace/skills/
├── gpt-researcher/
├── tavily-search/
├── memu/
└── ...
```

### 5.2 各 Agent 软链接

```bash
# 删除原有 skills 目录
rm -rf /root/ws-mime-qq/skills
rm -rf /root/ws-wife-qq/skills
rm -rf /root/ws-mime-feishu/skills

# 创建软链接
ln -s /root/.openclaw/workspace/skills /root/ws-mime-qq/skills
ln -s /root/.openclaw/workspace/skills /root/ws-wife-qq/skills
ln -s /root/.openclaw/workspace/skills /root/ws-mime-feishu/skills
```

**好处**：新增/更新 skill 只需操作一处，所有 Agent 自动生效。

## 八、总结

多渠道配置的核心：

1. **Agent 隔离** - 每个 Agent 独立 workspace
2. **绑定规则** - 明确消息路由
3. **记忆隔离** - 独立数据库 + 独立 memubot
4. **Skills 共享** - 软链接统一管理

配置完成后，三个场景互不干扰，各自积累独立的记忆和上下文。
