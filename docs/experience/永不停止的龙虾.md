# 永不停止的龙虾：让 OpenClaw 不中断工作指南

## 前言

"虾虾，帮我写一个完整的项目。"

三小时后，你回来看进度——

**"连接超时，请稍后重试。"**

或者更糟：

**"任务已完成！"** 

但你打开代码，发现只有一堆半截的函数。

这不是龙虾不够聪明，而是 **缺少"不中断"的机制**。

---

## 最简单的方法：直接问龙虾

### 操作步骤

**第一步**：告诉龙虾你的需求

```
我要实现一个【项目名称】。

这个项目比较复杂，可能需要多次对话才能完成。
请帮我规划一下如何保证任务中断后能顺利完成。
```

**第二步**：龙虾会给出方案

龙虾会自动帮你：
1. 拆解任务为小步骤
2. 建立进度追踪文件
3. 每完成一步就记录
4. 下次对话时先检查进度

**第三步**：后续对话自动恢复

下次对话时，只需要说：

```
继续上次的项目
```

龙虾会自动读取进度文件，从中断的地方继续。

## 进阶方法：配置 OpenClaw

如果你想让龙虾更"自觉"，可以配置 OpenClaw 的自动机制。

### 方法一：HEARTBEAT.md（定期检查）

**作用**：龙虾会定期检查是否有未完成的任务。

**配置方式**：

在项目根目录创建 `HEARTBEAT.md`：

```markdown
# HEARTBEAT.md

# 定期检查任务

每次心跳时检查：
1. 是否有未完成的项目？（检查 progress.md）
2. 是否需要主动通知用户？

如果有未完成任务，主动提醒用户。
```

**效果**：龙虾会定期（约 30 分钟）检查一次，如果有未完成任务，主动通知你。

### 方法二：MEMORY.md（长期记忆）

**作用**：让龙虾记住你的项目进度。

**配置方式**：

在项目根目录创建 `MEMORY.md`：

```markdown
# MEMORY.md

## 当前进行中的项目

### 用户认证模块
- **状态**：进行中
- **进度**：需求分析完成，数据库设计中
- **下一步**：完成数据库设计
- **文件**：`/project/auth/progress.md`

### OpenClaw Monitor
- **状态**：已完成
- **完成时间**：2026-03-15
```

**效果**：每次新对话，龙虾会自动读取 `MEMORY.md`，知道你在做什么。

### 方法三：定时提醒（qqbot-cron）

**作用**：让龙虾定时提醒你继续工作。

**使用方式**：

```
每天晚上8点提醒我继续用户认证模块
```

龙虾会设置定时提醒，到点主动提醒你。

## OpenClaw 自动保护机制

OpenClaw 本身有一些自动保护机制，你不需要配置，但了解一下有帮助：

### 1. Session 持久化

**作用**：对话历史自动保存。

**原理**：每次对话都会保存到 `.jsonl` 文件，下次对话自动加载。

**你不需要做什么**：这是 OpenClaw 默认行为。

### 2. 上下文自动管理

**作用**：长对话不会因为上下文溢出而中断。

**原理**：OpenClaw 会自动裁剪旧内容，保留关键信息。

**配置**（在 `openclaw.json`）：

```json
{
  "contextPruning": {
    "mode": "cache-ttl",
    "ttl": "1h",
    "keepLastAssistants": 3
  },
  "compaction": {
    "mode": "safeguard",
    "reserveTokens": 30000
  }
}
```

**你不需要做什么**：这是 OpenClaw 默认配置。

### 3. 模型降级链

**作用**：主模型不可用时，自动切换到备用模型。

**原理**：配置多个模型，失败时自动降级。

**配置**（在 `openclaw.json`）：

```json
{
  "model": {
    "primary": "tencentcodingplan/glm-5",
    "fallbacks": [
      "tencentcodingplan/hunyuan-2.0-thinking",
      "deepseek/deepseek-reasoner"
    ]
  }
}
```

**你不需要做什么**：这是 OpenClaw 默认配置。

## 原理简述

### 为什么龙虾会"失忆"？

**核心问题**：模型每一轮都在有限上下文中工作，每次执行都是一次"记忆重置"。

**表现**：
- Session 重置：每次对话都是新开始
- 上下文溢出：长任务中途截断
- 过早宣布完成：留下半成品

### 如何解决？

**核心思路**：把"记忆"存到文件里，每次对话先"恢复记忆"。

**OpenClaw 的三层记忆**：

| 层级 | 文件 | 用途 |
|------|------|------|
| **短期** | `sessions/*.jsonl` | 当前对话上下文 |
| **中期** | `memory/YYYY-MM-DD.md` | 每日笔记 |
| **长期** | `MEMORY.md` | 永久记忆 |

**工作流程**：

```
每次对话开始
    ↓
读取 MEMORY.md（长期记忆）
    ↓
读取 memory/YYYY-MM-DD.md（中期记忆）
    ↓
读取 sessions/*.jsonl（短期记忆）
    ↓
"恢复记忆"完成，开始工作
```
