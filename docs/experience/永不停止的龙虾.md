# 永不停止的龙虾

给龙虾一个复杂任务，先让它给出"不中断方案"。

---

## 操作流程

**第一步**：让龙虾给出方案

```
我要实现一个【项目名称】。

这个任务比较复杂，可能需要多次对话才能完成。
请给出方案：如何保证整个任务不被中断？
方案确认后，你再执行。
```

**第二步**：确认方案

龙虾会给出方案，比如：
- 任务拆解为哪些步骤
- 进度如何追踪
- 中断后如何恢复

你确认方案可行后，龙虾再执行。

**第三步**：中断后恢复

如果中断了，下次对话说"继续"。

龙虾会自动读取进度，从中断点继续。

---

## 实际案例

**用户**：
```
我要实现一个用户认证模块。
这个任务比较复杂，可能需要多次对话才能完成。
请给出方案：如何保证整个任务不被中断？
方案确认后，你再执行。
```

**龙虾回复**：
```
好的，我给出以下方案：

## 任务拆解
1. 需求分析
2. 数据库设计
3. API 接口设计
4. 核心代码实现
5. 测试用例编写
6. 文档编写

## 进度追踪
- 创建 `progress.md` 记录当前步骤
- 每完成一步更新状态

## 恢复机制
- 每次对话开始，读取 `progress.md`
- 从当前步骤继续

请确认方案是否可行？
```

**用户确认**：方案可行，开始执行。

---

## 进阶配置（可选）

### HEARTBEAT.md

让龙虾定期检查未完成任务，主动提醒你。

### MEMORY.md

让龙虾记住长期项目状态，跨会话持续工作。

### 模型降级链

主模型失败时自动切换备用模型，减少中断。

配置示例（`openclaw.json`）：

```json
{
  "model": {
    "primary": "tencentcodingplan/glm-5",
    "fallbacks": [
      "tencentcodingplan/hunyuan-2.0-thinking",
      "deepseek/deepseek-reasoner"
    ]
  }
}
```

---

## 核心原则

**先问方案**：让龙虾给出不中断方案，确认后再执行。

**中断恢复**：说"继续"，龙虾自动恢复。

**配置可选**：HEARTBEAT、MEMORY、降级链都是锦上添花。
