# 永不停止的龙虾

龙虾能自主跟踪任务直到完成，无需操心。

真正中断时（网络故障、会话超时），才需要恢复机制。

---

## 最简单的方法

开始新项目时，直接说：

```
我要实现一个【项目名称】。
请帮我规划进度追踪方式，确保中断后能恢复。
```

龙虾会自动：
1. 拆解任务
2. 建立进度文件
3. 每步记录状态

后续只需说"继续"，龙虾自动从中断点继续。

---

## 进阶配置

### HEARTBEAT.md

让龙虾定期检查未完成任务。

在项目根目录创建：

```markdown
# HEARTBEAT.md

每次心跳检查：
- 是否有未完成的项目？
- 是否需要主动通知？
```

### MEMORY.md

让龙虾记住项目状态。

在项目根目录维护：

```markdown
# MEMORY.md

## 当前进行中的项目

### 用户认证模块
- 状态：进行中
- 进度：数据库设计中
- 下一步：完成 API 接口
```

### 定时提醒

让龙虾定时提醒你：

```
每天晚上8点提醒我继续用户认证模块
```

## 自动保护机制

OpenClaw 默认提供：

| 机制 | 作用 |
|------|------|
| Session 持久化 | 对话历史自动保存 |
| 上下文管理 | 长对话自动裁剪 |

需要手动配置：

| 机制 | 作用 |
|------|------|
| 模型降级链 | 主模型失败自动切换备用 |

**降级链配置示例**（`openclaw.json`）：

```json
{
  "model": {
    "primary": "tencentcodingplan/glm-5",
    "fallbacks": [
      "tencentcodingplan/hunyuan-2.0-thinking",
      "deepseek/deepseek-reasoner"
    ]
  }
}
```

## 核心原则

**信任龙虾**：它能自主跟踪直到完成，无需干预。

**记录进度**：中断后说"继续"，龙虾会自动恢复。

**配置可选**：HEARTBEAT、MEMORY、降级链都是锦上添花。
