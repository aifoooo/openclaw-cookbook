# 长期的伙伴：让龙虾拥有长期记忆

OpenClaw 的 MEMORY.md 机制是一种「被动记忆」——文件持久化，但龙虾不会主动判断什么值得记住。

我想要「主动记忆」——龙虾自己判断什么该记，自动保存，下次见面还能想起来。

这篇文章记录了给龙虾装上长期记忆的方案。

---

## 一、为什么需要长期记忆

即使配置了 14 天空闲重置，仍然有问题：

1. 对话崩了，`/new` 新建会话，龙虾不记得之前聊了什么
2. 很久前提的事，龙虾早就忘了
3. 用户偏好散落在各次对话中，无法整合

**一句话**：没有长期记忆，每次对话都是从零开始。

---

## 二、方案选型

Google 的 [Always On Memory Agent](https://github.com/GoogleCloudPlatform/generative-ai/tree/main/gemini/agents/always-on-memory-agent) 提供了一个思路：**模拟人脑的记忆整合机制**——睡眠时回放经历，发现关联，压缩成稳定记忆。

GitHub 上有几个现成方案：

| 项目 | Stars | 特点 |
|-----|-------|------|
| mem0 | 49k+ | Y Combinator 孵化，通用记忆层 |
| memU | 12k+ | **专为 OpenClaw 设计** |
| Letta | 21k+ | 前身 MemGPT，自进化记忆 |
| memvid | 13k+ | 单文件存储，无需数据库 |

**选择 memU**：README 直接提到 OpenClaw，专为 24/7 运行的 Agent 设计，支持主动记忆机制。

---

## 三、memU 原理

memU 的核心是「记忆整合」：

```
对话 → LLM 提取记忆项 → Embedding 向量化 → PostgreSQL 存储
                                    ↓
                         需要时向量相似度搜索
```

**三个组件协作**：

1. **LLM（Chat API）**：从对话中提取记忆项（偏好、事实、目标、事件）
2. **Embedding 模型**：将记忆项向量化，用于相似度搜索
3. **PostgreSQL + pgvector**：持久化存储，支持向量索引

---

## 四、实施方案

### 4.1 架构

```
OpenClaw 主会话
    │
    ├─▶ MEMORY.md（核心记忆，手动维护）
    │
    └─▶ sessions/*.jsonl（对话记录）
            │
            ▼ memUbot 守护进程（每 10 秒同步）
        长期记忆系统
            │
            ├─ 腾讯云 GLM-5：提取记忆项
            │
            ├─ 本地 bge-small-zh-v1.5：向量化（~0.1秒）
            │
            └─ PostgreSQL + pgvector：存储
```

**选择理由**：
- **Chat API 用腾讯云**：GLM-5 延迟约 1 秒，可接受
- **Embedding 用本地模型 bge-small-zh-v1.5**：免费、快速（~0.1秒）、中文效果好
- **PostgreSQL + pgvector**：官方推荐，支持向量索引

### 4.2 memUbot 守护进程

memUbot 是一个 systemd 服务，每 10 秒自动同步新对话：

```bash
# 启动服务
systemctl start memubot

# 查看状态
systemctl status memubot

# 查看日志
journalctl -u memubot -f
```

**工作流程**：

1. 每 10 秒扫描会话文件，找到新对话
2. 调用 GLM-5 提取记忆项
3. 本地 Embedding 向量化（~0.1秒）
4. 存入 PostgreSQL（自动去重）

**去重机制**：
- 消息 ID 去重：启动时加载已同步消息，避免重复处理
- 向量去重：存储时检查相似度，>90% 视为重复跳过

### 4.3 新会话自动加载记忆

在 `AGENTS.md` 中配置：

```markdown
## Every Session

4. **查询最近 10 条记忆**：执行 `python local_memory.py recent 10`
```

**效果**：新会话自动了解用户偏好和历史。

## 五、效果验证

以下是 memU 在实际使用中的案例：

### 案例一：跨会话记忆恢复

**场景**：用户执行 `/new` 新建会话后，询问之前的技能

**用户问**：我们之前不是写了一个写文章的 skill 吗，记得吗？

**memU 返回**：
- 技能名称：writing-pipeline
- 位置：`/root/.openclaw/workspace/skills/writing-pipeline/`
- 三种场景：新建文章、优化文章、快速观点文

**实际回复**：记得。`writing-pipeline` skill，三种场景...

✅ 跨会话记忆恢复，技能信息不丢失

### 案例二：项目上下文记忆

**场景**：询问项目状态

**用户问**：我们之前在写 cookbook，你记得吗？

**memU 返回**：
- 仓库位置：`/root/.openclaw/workspace/openclaw-cookbook`
- 已完成文章：《龙虾初相遇》《深度调研》《长期的伙伴》
- 目录结构：experience/、learning/、changelog/

**实际回复**：记得。`openclaw-cookbook` 仓库，三篇文章...仓库已推送到 GitHub

✅ 项目信息跨会话可用

### 案例三：用户偏好记忆

**场景**：新会话开始，memU 自动加载用户偏好

**memU 自动加载**：
- 用户背景：Java 开发工程师
- 核心准则：绝对不臆造、穷尽方法解决问题、我能做的事情就不让用户做
- 沟通方式：通过 QQ 私聊

**效果**：新会话自动了解用户偏好，无需重复说明

✅ 偏好自动从对话中提取并持久化
