# 长期的伙伴

即使配置了 14 天空闲重置，仍有问题：对话崩了不记得之前聊了什么，用户偏好散落在各次对话中无法整合。

以下是给龙虾装上长期记忆的方案。

---

## 一、方案选型

即使配置了 14 天空闲重置，仍有问题：对话崩了不记得之前聊了什么，用户偏好散落在各次对话中。

**解决方案**：安装 memU 长期记忆系统。

| 项目 | Stars | 特点 |
|-----|-------|------|
| mem0 | 49k+ | Y Combinator 孵化，通用记忆层 |
| memU | 12k+ | **专为 OpenClaw 设计** |
| Letta | 21k+ | 前身 MemGPT，自进化记忆 |
| memvid | 13k+ | 单文件存储，无需数据库 |

**选择 memU**：README 直接提到 OpenClaw，专为 24/7 运行的 Agent 设计。

---

## 二、memU 原理

```
对话 → LLM 提取记忆项 → Embedding 向量化 → PostgreSQL 存储
                                    ↓
                         需要时向量相似度搜索
```

**三个组件**：

| 组件 | 作用 |
|------|------|
| LLM（Chat API） | 从对话中提取记忆项（偏好、事实、目标、事件） |
| Embedding 模型 | 将记忆项向量化，用于相似度搜索 |
| PostgreSQL + pgvector | 持久化存储，支持向量索引 |

---

## 三、实施方案

### 3.1 架构

```
OpenClaw 主会话
    │
    ├─▶ MEMORY.md（核心记忆，手动维护）
    │
    └─▶ sessions/*.jsonl（对话记录）
            │
            ▼ memUbot 守护进程（每 10 秒同步）
        长期记忆系统
            │
            ├─ 腾讯云 GLM-5：提取记忆项
            │
            ├─ 本地 bge-small-zh-v1.5：向量化（~0.1秒）
            │
            └─ PostgreSQL + pgvector：存储
```

### 3.2 memUbot 守护进程

memUbot 是一个 systemd 服务，每 10 秒自动同步新对话。

```bash
# 启动服务
systemctl start memubot

# 查看状态
systemctl status memubot

# 查看日志
journalctl -u memubot -f
```

**工作流程**：

1. 每 10 秒扫描会话文件，找到新对话
2. 调用 GLM-5 提取记忆项
3. 本地 Embedding 向量化（~0.1秒）
4. 存入 PostgreSQL（自动去重）

### 3.3 新会话自动加载记忆

在 `AGENTS.md` 中配置：

```markdown
## Every Session

4. **查询最近 10 条记忆**：执行 `python local_memory.py recent 10`
```

**效果**：新会话自动了解用户偏好和历史。

## 四、效果验证

### 案例一：跨会话记忆恢复

**场景**：用户执行 `/new` 新建会话后，继续之前的讨论

**之前对话**：用户在讨论博客项目应该如何命名，倾向于用 `learning` 作为学习笔记目录名

**新会话问**：我之前说的目录名用哪个？

**memU 返回**：用户偏好使用 `learning` 命名学习笔记目录

**实际回复**：你之前说用 `learning` 作为学习笔记目录名...

### 案例二：项目上下文记忆

**场景**：询问项目状态

**用户问**：我们之前在写 cookbook，你记得吗？

**memU 返回**：
- 仓库位置：`/root/.openclaw/workspace/openclaw-cookbook`
- 已完成文章：《龙虾初相遇》《深度调研》《长期的伙伴》
- 目录结构：experience/、learning/、changelog/

**实际回复**：记得。`openclaw-cookbook` 仓库，三篇文章...仓库已推送到 GitHub

### 案例三：用户偏好记忆

**场景**：新会话开始，memU 自动加载用户偏好

**memU 自动加载**：
- 用户背景：Java 开发工程师
- 核心准则：绝对不臆造、穷尽方法解决问题、我能做的事情就不让用户做
- 沟通方式：通过 QQ 私聊

**效果**：新会话自动了解用户偏好，无需重复说明
