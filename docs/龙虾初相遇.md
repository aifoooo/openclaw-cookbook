# 龙虾初相遇

新装的 OpenClaw 就像刚领回家的宠物——用起来各种不顺手，得磨合一阵子才会"懂你"。

这篇文章记录了我与龙虾从陌生到默契的过程，希望帮后来者少走弯路。

---

## 一、开眼看世界

新安装的 OpenClaw 默认没有联网能力，问啥啥不知。

**解决方案**：让龙虾帮你配置 Tavily。

只需要跟龙虾说"需要安装 Tavily"，它会自动安装，最后问你要 API Key。按提示去 Tavily 官网注册、获取 Key，丢给它就行。

> 💡 **一句话**：联网能力是龙虾的"眼睛"，没有它就是井底之蛙。

---

## 二、告别"傻"龙虾

刚开始，龙虾经常胡说八道，动不动就说"我不会"，动不动就把任务推给我。

我把下面的核心准则告诉它，让它务必永久遵循——它就像换了个灵魂似的，变得靠谱多了。

**核心准则：**

1. **绝对不臆造** — 不确定就先核实，所有输出基于可验证事实
2. **穷尽方法解决问题** — 不轻易说"做不到"，有限制就提供替代方案
3. **我能做的事情就不让用户做** — 凡是我能完成的任务，绝不推给用户
4. **犯过的错绝不再犯** — 从错误中学习，避免重复踩坑
5. **永久遵循** — 以上准则贯穿所有交互

> 💡 **一句话**：核心准则就是龙虾的"价值观"，价值观对了，行为就对了。

---

## 三、突然失忆

第一天聊得好好的，第二天它就忘记了昨天的事——很郁闷。

**原因**：OpenClaw 默认在每天凌晨 4 点重置会话。

**解决方案**：配置空闲重置逻辑，14 天不活跃才重置。

```json
"session": {
  "reset": {
    "mode": "idle",
    "idleMinutes": 20160
  }
}
```

**但还不够**：聊多了会突然崩掉，崩后 `/new` 恢复，但聊天记录全丢。

**保险做法**：时不时让它把重要内容压缩存储到 MEMORY.md。就像写文档时的"定时保存"——崩了也能找回。

> 💡 **一句话**：龙虾的记忆是易失的，重要的事情要写到文件里。

---

## 四、上下文溢出

跟龙虾聊多了，经常会遇到"上下文超出大模型限制"的问题。

**解决方案**：配置压缩和剪枝策略。

```json
"compaction": {
  "mode": "safeguard",
  "reserveTokens": 60000,
  "keepRecentTokens": 80000,
  "memoryFlush": {
    "enabled": true,
    "softThresholdTokens": 15000
  }
},
"contextPruning": {
  "mode": "cache-ttl",
  "ttl": "1h",
  "keepLastAssistants": 3,
  "softTrimRatio": 0.3,
  "hardClearRatio": 0.5
}
```

> 💡 **一句话**：上下文就像内存，满了就得清理，但要保留核心数据。

---

## 五、模型降级

某个模型调用次数超限，或者欠费了——怎么办？

**解决方案**：配置降级链，让它自动切换到备用模型。

```json
"model": {
  "primary": "tencentcodingplan/glm-5",
  "fallbacks": [
    "tencentcodingplan/hunyuan-2.0-thinking",
    "deepseek/deepseek-reasoner"
  ]
}
```

> 💡 **一句话**：模型降级就是"备胎策略"，主挂了有备用的顶上。

---

## 六、永久的能力

安装了某个工具，龙虾不一定时时记得。聊着聊着，它就忘了曾经安装的软件、曾经具备的能力。

**解决方案**：把能力封装成 skill。

PDF 转换、深度研究报告生成、视频截图提取……都可以封装成 skill。一旦封装，龙虾就永久拥有这些能力。

> 💡 **一句话**：skill 就是龙虾的"技能树"，点亮了就不会忘记。

---

## 写在最后

上面的配置大多不用自己动手。跟龙虾聊一下，提出要求，让它知道你要什么——它会帮你搞定。

OpenClaw 的魅力在于：**它不是一成不变的工具，而是会成长的伙伴**。你花时间调教它，它会越来越懂你。

> 🦐 **驯服一只龙虾，需要耐心，但值得。**
