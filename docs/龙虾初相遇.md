# 龙虾初相遇

新装的 OpenClaw 会有各种各样的问题，用起来各种不顺手。经过一段时间的摸索、磨合后，它才会变得"更懂你"。

---

## 一、开眼看世界

新安装的 OpenClaw 会认为自己没有联网能力，问啥啥不知。

这个时候可以让龙虾帮忙配置 Tavily。只需要跟龙虾说需要安装 Tavily，龙虾会自动安装，最后问我们要 API Key，按它的提示获取后丢给它就行。

---

## 二、告别"傻"龙虾

刚开始，龙虾经常胡说八道，动不动就说它不会，动不动就把任务推给我。

我把下面的核心准则告诉它，让它务必永久遵循，它就像换了个灵魂似的乖乖听话。

**核心准则：**

1. **绝对不臆造** — 不确定就先核实，所有输出基于可验证事实
2. **穷尽方法解决问题** — 不轻易说"做不到"，有限制就提供替代方案
3. **我能做的事情就不让用户做** — 凡是我能完成的任务，绝不推给用户
4. **永久遵循** — 以上准则贯穿所有交互

---

## 三、突然失忆

第一天跟龙虾聊得好好的，第二天它就忘记了昨天的事，很郁闷。

和它探讨了一阵子后，我明白了：它会在每天凌晨 4 点重置会话，这就是它"失忆"的原因。

于是，我让它配置了 14 天空闲再重置的逻辑。

当然，这样的配置也不保险。跟它聊多了，有时候会突然崩掉。崩掉后如何恢复？很多时候我会在对话框输入 `/new`，它会重置会话恢复起来，但之前的聊天记录就全部丢了。

所以，保险的做法是：时不时让它把聊天内容压缩后存储到 MEMORY.md。这样，它就能在崩溃恢复之后，还记得之前的聊天内容。（类似编写文档时的定时保存）

```json
"session": {
  "reset": {
    "mode": "idle",
    "idleMinutes": 20160
  }
}
```

---

## 四、上下文溢出

跟龙虾聊多了，经常会遇到上下文超出大模型限制的问题，可以通过下面的配置减少类似问题的出现。

```json
"compaction": {
  "mode": "safeguard",
  "reserveTokens": 60000,
  "keepRecentTokens": 80000,
  "memoryFlush": {
    "enabled": true,
    "softThresholdTokens": 15000
  }
},
"contextPruning": {
  "mode": "cache-ttl",
  "ttl": "1h",
  "keepLastAssistants": 3,
  "softTrimRatio": 0.3,
  "hardClearRatio": 0.5
}
```

---

## 五、模型降级

如果某个模型调用次数超过限制，或者欠费了，可以让龙虾自动切换到其他模型。

```json
"model": {
  "primary": "tencentcodingplan/glm-5",
  "fallbacks": [
    "tencentcodingplan/hunyuan-2.0-thinking",
    "deepseek/deepseek-reasoner"
  ]
}
```

---

## 六、永久的能力

安装了某个工具，龙虾不一定时时记得。有时候聊着聊着，它就忘了曾经安装的软件、曾经具备的能力。

这个时候可以把这些能力让龙虾封装成 skill，它就永久拥有这些能力了。

比如 PDF 转换、深度研究报告生成、视频截图提取，都可以封装成 skill。

---

上面的很多配置都不用我们自己动手，基本上跟龙虾聊一下，提出要求，让它知道这个事情，它就会帮我们配置好。
