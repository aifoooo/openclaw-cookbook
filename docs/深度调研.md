# 深度调研：让龙虾帮你写研究报告

有时候想深入了解某个主题，大模型返回的内容可靠度往往不够——太浅、没数据、没来源。

这时候可以让龙虾调用 GPT-Researcher，抓取 20+ 网页，生成一份结构化的研究报告。

---

## 一、GPT-Researcher 是什么

GPT-Researcher 是一个开源的 AI 研究助手，能够：

- 自动搜索并抓取多个网页来源
- 过滤和整理信息
- 生成结构化的研究报告
- 保留所有来源引用

**一句话**：它就像一个不知疲倦的研究员，帮你把网上散落的信息拼成一份报告。

---

## 二、安装与配置

### 安装

跟龙虾说"帮我安装 GPT-Researcher"，它会自动完成。

### 配置

安装后，需要配置 API Keys。配置文件位置：`/root/gpt-researcher/.env`

```bash
# 腾讯云 GLM-5 (OpenAI 兼容)
OPENAI_API_KEY=你的腾讯云API密钥
OPENAI_BASE_URL=https://api.lkeap.cloud.tencent.com/coding/v3

# Tavily 搜索
TAVILY_API_KEY=你的Tavily密钥

# LLM 配置
FAST_LLM=openai:glm-5
SMART_LLM=openai:glm-5
STRATEGIC_LLM=openai:glm-5

# Embedding - HuggingFace 本地模型
EMBEDDING=huggingface:sentence-transformers/all-MiniLM-L6-v2

# 搜索配置
RETRIEVER=tavily

# 语言
LANGUAGE=chinese
```

> 💡 **为什么用 HuggingFace 本地模型？** 腾讯云暂无 Embedding API，本地模型免费且够用。

## 三、使用方式

### 方式一：直接对话

跟龙虾说：
```
帮我调研一下 2026 年 AI Agent 记忆系统的发展趋势
```

它会自动调用 GPT-Researcher，等待 2-3 分钟，返回一份报告摘要。

### 方式二：指定报告类型

```
帮我做一个资源汇总：广州增城区社工转行推荐岗位
```

报告类型：
- `research_report` — 综合研究报告（默认）
- `resource_report` — 资源汇总
- `outline_report` — 结构化大纲

## 四、上下文溢出问题

### 问题

GPT-Researcher 返回的内容很大（10000+ 字），加上平时的聊天内容，很容易超出大模型的上下文限制：

```
Context limit exceeded. I've reset our conversation to start fresh - please try again.
```

### 解决方案

让龙虾写一个脚本，把 GPT-Researcher 的输出先压缩成摘要，再返回给用户。

核心思路：
```
完整报告 → 保存到文件 → LLM 压缩成 3000 字摘要 → 返回摘要
```

这样既保留了完整报告（可查看），又不会撑爆上下文。

> 💡 **一句话**：大内容先存文件，只返回摘要——这是处理"大输出"的通用策略。

## 五、效果展示

**输入**：
```
帮我调研一下 OpenClaw 官方和社区提供的 Skills 技能库
```

- ClawHub：官方市场，5700+ 技能
- awesome-openclaw-skills：社区精选，2868 个高质量技能
- 安装方式：`npx clawhub@latest install <技能名>`
- 安全提醒：曾发生 ClawHavoc 供应链投毒，需审计源码
- 推荐技能：skill-vetter、tavily-search、self-improving-agent

**完整报告**：自动保存到 `reports/research_20260312_*.md`
