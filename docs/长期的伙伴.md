# 长期的伙伴：让龙虾拥有长期记忆

OpenClaw 的 MEMORY.md 机制是一种「被动记忆」——文件持久化，但龙虾不会主动判断什么值得记住。

我想要「主动记忆」——龙虾自己判断什么该记，自动保存，下次见面还能想起来。

这篇文章总结了给龙虾装上长期记忆的方案和最佳实践。

---

## 一、为什么需要长期记忆

即使配置了 14 天空闲重置，仍然有问题：

1. 对话崩了，`/new` 新建会话，龙虾不记得之前聊了什么
2. 很久前提的事，龙虾早就忘了
3. 用户偏好散落在各次对话中，无法整合

**一句话**：没有长期记忆，每次对话都是从零开始。

---

## 二、方案选型

Google 的 [Always On Memory Agent](https://github.com/GoogleCloudPlatform/generative-ai/tree/main/gemini/agents/always-on-memory-agent) 提供了一个思路：**模拟人脑的记忆整合机制**——睡眠时回放经历，发现关联，压缩成稳定记忆。

GitHub 上有几个现成方案：

| 项目 | Stars | 特点 |
|-----|-------|------|
| mem0 | 49k+ | Y Combinator 孵化，通用记忆层 |
| memU | 12k+ | **专为 OpenClaw 设计** |
| Letta | 21k+ | 前身 MemGPT，自进化记忆 |
| memvid | 13k+ | 单文件存储，无需数据库 |

**选择 memU**：README 直接提到 OpenClaw，专为 24/7 运行的 Agent 设计，支持主动记忆机制。

## 三、最佳实践

### 3.1 本地化 Embedding 模型

**问题**：云端 Embedding API 延迟高（~11秒），成本累积

**解决方案**：本地部署 bge-small-zh-v1.5

```python
from sentence_transformers import SentenceTransformer

model = SentenceTransformer('BAAI/bge-small-zh-v1.5')
embedding = model.encode(text)  # ~0.1秒，免费
```

**性能对比**：

| 项目 | 云端 API | 本地模型 |
|------|---------|---------|
| 延迟 | ~11秒 | ~0.1秒 |
| 成本 | 按量付费 | 免费 |
| 中文效果 | 一般 | 更好 |

**建议**：优先使用本地模型，Chat API 仍可用云端（腾讯云 GLM-5 延迟约 1 秒，可接受）

### 3.2 PostgreSQL + pgvector 存储

**建议**：直接用 PostgreSQL + pgvector，跳过 SQLite

原因：
- SQLite 有 SQLModel 版本兼容性问题
- pgvector 支持向量相似度搜索，是官方推荐方案

```sql
CREATE TABLE memories (
    id SERIAL PRIMARY KEY,
    memory_type VARCHAR(50),
    content TEXT,
    importance INTEGER,
    embedding vector(512),
    created_at TIMESTAMP DEFAULT NOW()
);
```

### 3.3 定时同步，而非实时

**建议**：Cron 定时同步，而非每次对话后实时写入

原因：
- 实时写入增加对话延迟
- 批量处理更高效

实现：
```bash
# 每分钟检查新对话并同步
* * * * * cd /root/.openclaw/workspace/skills/memu && python sync.py
```

### 3.4 新会话自动加载记忆

**建议**：每次新会话自动查询最近 N 条记忆

在 `AGENTS.md` 中加入：
```markdown
## Every Session

4. **查询最近 10 条记忆**：执行 `python local_memory.py recent 10`
```

**效果**：新会话自动了解用户偏好和历史，无需手动查询

### 3.5 记忆去重

**问题**：Cron 任务重复同步同一条对话

**解决方案**：维护同步状态文件，启动时加载一次，持续更新

```python
# 加载已处理的对话 ID
processed_ids = load_state()

# 同步后更新状态
save_state(processed_ids)
```

## 四、最终架构

```
OpenClaw 主会话
    │
    ├─▶ MEMORY.md（核心记忆，手动维护）
    │
    └─▶ session-memory（对话记录）
            │
            ▼ Cron 每分钟同步
        memU（长期记忆系统）
            │
            ├─ memorize：LLM 提取记忆项 + Embedding 向量化
            │
            └─ PostgreSQL + pgvector：持久化存储
```

**工作流程**：

1. **对话记录**：每次对话写入 `sessions/YYYY-MM-DD.jsonl`
2. **定时同步**：Cron 每分钟检查新对话
3. **记忆提取**：LLM 提取偏好、画像、重要事件
4. **向量化存储**：Embedding + PostgreSQL
5. **查询记忆**：需要时向量相似度搜索

## 五、效果验证

### 场景一：会话重置后还记得

告诉龙虾：*我最近在学 Rust，用 IntelliJ IDEA 写代码*

`/new` 后问：**我最近在学什么语言？**

答：**根据记忆，你最近在学 Rust。**

✅ 跨会话记忆生效

### 场景二：以前的事也能想起来

之前提过 openclaw-cookbook 项目。新会话问：**我最近在做什么项目？**

答：**你在做 openclaw-cookbook，是 OpenClaw 使用经验的博客仓库。**

✅ 向量相似度搜索有效

### 场景三：偏好自动提取

从没明确说过偏好，只是随口提了一句编辑器。

问：**你知道我有哪些偏好吗？**

答：**你喜欢用 IntelliJ IDEA 写代码。**

✅ 自动从对话中提取偏好

## 六、写在最后

**但已经足够好用**——龙虾不再是"金鱼记忆"，而是能记住偏好、项目、重要事件。
