# 长期的伙伴

## 一、记忆系统的缺陷

我已经配置了 OpenClaw 在空闲 14 天后才重置对话，但这样的设置仍然无法满足我的诉求：

1. 对话崩了，通过 `/new` 新建会话，龙虾并不记得之前聊了什么
2. 提及很久之前聊过的问题，龙虾往往已经忘了，除非当时明确让它记住

这让我意识到：OpenClaw 自带的 MEMORY.md 机制，本质上是一种「被动记忆」——文件本身是持久化的，会话重置后还在。但问题是，MEMORY.md 的内容很多时候是我明确让龙虾「记住这个」，它才会写入。龙虾没有主动性，不会自己判断哪些信息有价值、值得长期保存。

我想要的是「主动记忆」——龙虾能自己判断什么值得记住，自动保存，下次见面还能想起来。

## 二、外挂记忆系统

Google 在 GitHub 上开源的 [Always On Memory Agent](https://github.com/GoogleCloudPlatform/generative-ai/tree/main/gemini/agents/always-on-memory-agent) 提到了让 Agent 拥有长期记忆的一个思路：

**模拟人脑的记忆整合机制**。

人脑在睡眠时会回放白天的经历，发现事件之间的关联，把零散的信息压缩成稳定的记忆。这个项目设计了三个 Agent 协作：

- IngestAgent：接收对话，提取关键信息
- ConsolidateAgent：定时整合，发现记忆之间的联系
- QueryAgent：综合记忆，回答问题

核心思想是：记忆不是简单的存储，而是一个持续处理、整合的过程。

这个思路很棒，但我需要的是现成的方案，而不是从头造轮子。于是我让龙虾调研了一圈 GitHub 上的开源项目。

**调研结果**：

| 项目 | Stars | 特点 |
|-----|-------|------|
| mem0 | 49k+ | Y Combinator 孵化，通用记忆层 |
| memU | 12k+ | **专为 OpenClaw 设计** |
| Letta | 21k+ | 前身 MemGPT，自进化记忆 |
| memvid | 13k+ | 单文件存储，无需数据库 |

memU 的 README 直接提到 OpenClaw，这让我眼前一亮——专门为 OpenClaw 这类 24/7 运行的 Agent 设计，支持「主动记忆」机制。就是它了。

## 三、龙虾的实践

把 memU 跑起来，过程比预想的曲折。

### 3.1 第一个坑：腾讯云 Embedding 配置失败

memU 需要 Chat API + Embedding API 配合使用：
- Chat API：提取记忆项
- Embedding API：向量化，用于相似度搜索

我使用的主力 LLM 是腾讯云 GLM-5。龙虾尝试配置腾讯云混元 Embedding API，但发现 memU 原生不支持腾讯云的签名认证方式，魔改成本较高。

龙虾提供了两个方案：
1. 直接换阿里云全家桶（Chat + Embedding）
2. 魔改 memU，用本地 sentence-transformers

我选了方案 1——阿里云全家桶，简单省事。

### 3.2 第二个坑：SQLite 持久化失败

memU 默认用 SQLite 存储记忆，但遇到了 SQLModel 版本兼容性问题。龙虾建议改用 PostgreSQL——memU 官方推荐的方案。

于是又折腾了一番：
- 安装 PostgreSQL 15
- 安装 pgvector 扩展（向量搜索）
- 创建 memu 数据库和用户

这次终于成功了。

### 3.3 第三个坑：网络延迟

阿里云 API 在北京，我的服务器在香港。跨区域调用延迟较高，而 memU 的 memorize 操作需要多次 API 调用（提取记忆项 + 向量化），一次完整的 memorize 可能要 30-60 秒。默认的超时设置经常报错。

龙虾把超时设置从默认的 30 秒调到了 120 秒，暂时能用了。后续可以换成腾讯云混元 Embedding（同区域延迟低），或者本地 sentence-transformers。

### 3.4 最终架构

```
┌─────────────────────────────────────────────────────┐
│                    OpenClaw                          │
│  ┌──────────────┐    ┌──────────────────────────┐   │
│  │  主会话       │───▶│  MEMORY.md（核心记忆）    │   │
│  └──────────────┘    └──────────────────────────┘   │
│         │                                            │
│         ▼                                            │
│  ┌──────────────────────────────────────────────┐   │
│  │  session-memory（对话记录）                    │   │
│  │  sessions/YYYY-MM-DD.jsonl                   │   │
│  │  每条记录：{role, content, timestamp}         │   │
│  └──────────────────────────────────────────────┘   │
│         │                                            │
│         ▼ Cron（每分钟）                              │
│  ┌──────────────────────────────────────────────┐   │
│  │  sync_to_memu.py                              │   │
│  │  1. 读取 jsonl 中新增的对话                    │   │
│  │  2. 调用 memU memorize                        │   │
│  │  3. LLM 提取记忆项（偏好、画像等）             │   │
│  │  4. Embedding 向量化                          │   │
│  │  5. 存入 PostgreSQL                          │   │
│  └──────────────────────────────────────────────┘   │
│         │                                            │
│         ▼                                            │
│  ┌──────────────────────────────────────────────┐   │
│  │  memU（长期记忆系统）                          │   │
│  │  ├─ memorize：自动提取记忆项 + 向量化存储      │   │
│  │  ├─ retrieve：向量相似度搜索                   │   │
│  │  └─ PostgreSQL + pgvector：持久化存储         │   │
│  └──────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────┘
```

**工作流程**：

1. **对话记录**：OpenClaw 每次对话都会写入 `sessions/YYYY-MM-DD.jsonl`
2. **定时同步**：Cron 每分钟检查是否有新对话
3. **记忆提取**：调用 memU 的 memorize，LLM 从对话中提取有价值的记忆项（用户偏好、画像、重要事件等）
4. **向量化存储**：Embedding API 将记忆项向量化，存入 PostgreSQL
5. **查询记忆**：需要时通过 retrieve 进行向量相似度搜索

**触发时机**：

- **记忆写入**：当前通过 Cron 定时同步（每分钟）；理想状态是每次对话结束后实时提取
- **记忆查询**：当前需要手动调用 retrieve；理想状态是每次对话前自动检索相关记忆，注入到上下文中（需要魔改 OpenClaw 核心）

## 四、胜利在望

memU 跑起来了，来测试一下核心场景。

### 4.1 场景一：会话重置后还记得

我在对话中告诉龙虾：
> 我最近在学 Rust，用 IntelliJ IDEA 写代码

聊了几句别的事，然后发送 `/new` 新建会话。

新会话里，我问：
> 我最近在学什么语言？

龙虾回答：
> 根据记忆，你最近在学 Rust。

成功了！memU 的记忆跨会话生效。

### 4.2 场景二：以前的事也能想起来

不久前我提过自己在做 openclaw-cookbook 项目。新建会话后，我问：
> 我最近在做什么项目？

龙虾回答：
> 你在做 openclaw-cookbook，是一个 OpenClaw 使用经验的博客仓库，已经写了两篇文章：《龙虾初相遇》和《深度调研》。

向量相似度搜索找到了相关记忆，即使那是以前的对话。

### 4.3 场景三：偏好自动提取

我从没明确告诉龙虾「我喜欢 IntelliJ IDEA」，只是在它问我用什么编辑器时随口说了一句。

但后来问龙虾：
> 你知道我有哪些偏好吗？

龙虾回答：
> 你喜欢用 IntelliJ IDEA 写代码。

memU 自动从对话中提取了这些偏好，无需我专门说「记住这个」。

### 4.4 还差一步

目前 memU 的记忆写入和查询都需要手动触发（或通过 Cron 定时同步）。理想状态是：

- 每次对话结束，自动判断是否有值得记住的信息
- 每次对话开始，自动检索相关记忆注入上下文

这需要魔改 OpenClaw 的核心流程，暂时先这样用着。
