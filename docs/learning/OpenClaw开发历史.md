# OpenClaw 开发历史：从周末项目到 GitHub 榜首

> 4 个月，18 万星标，增长曲线近乎垂直。这不是营销神话，而是一个"独狼开发者"搅动整个 AI 产业的故事。

## 引言：开源史上的罕见现象

2026 年 2 月，OpenClaw 登顶 GitHub 榜首。

这个成绩的惊人之处不在于数字本身，而在于时间——从 2025 年 11 月诞生到登顶，只用了 4 个月。作为对比，React 达到同样高度用了 13 年。

**这不是营销驱动的增长，而是技术共鸣的爆发。**

---

## 一、创始人：Peter Steinberger

### 1.1 背景

Peter Steinberger，奥地利开发者：

| 经历 | 详情 |
|------|------|
| 2011 年 | 创办 PSPDFKit（iOS PDF 开发工具） |
| 产品覆盖 | 10 亿+ 设备 |
| 2021 年 | 获得 1.16 亿美元投资后退出 |
| 2021-2024 | 经历三年职业倦怠 |
| 2024 年 | AI 浪潮重燃热情，开始 tinkering |

### 1.2 周末原型

2025 年 11 月的一个周末，Steinberger 用大约 1 小时完成原型：

- 将 WhatsApp 与 Claude Code 连接
- 创建能自主执行任务的 AI 智能体
- 起名 Clawdbot（Claude + Claw 双关）

**他原本只是想写个小工具，结果一不小心写成了一个完整的 AI 自主智能体。**

## 二、爆发式增长

### 2.1 数据

| 指标 | 数据 |
|------|------|
| GitHub 星标 | 4 个月从 0 到 18 万+ |
| 对比参照 | React 用了 13 年 |
| 周访问量 | 200 万访客 |
| 增长曲线 | 近乎垂直 |

### 2.2 为什么火？

**关键洞察**：把程序员专属工具推向非技术用户。

| 之前 | 之后 |
|------|------|
| Cursor/Claude Code 只有程序员用 | 接入 WhatsApp/飞书/钉钉 |
| 学习成本高 | "拿起手机就能用" |
| 技术壁垒 | 零门槛 |

**本质**：把一小撮人享受的能力，第一次推到了更广泛的人群面前。

## 三、关键里程碑

### 3.1 更名风波

| 时间 | 名称 | 原因 |
|------|------|------|
| 初期 | Clawdbot | 致敬 Claude |
| 第一次更名 | Moltbot | Anthropic 商标投诉 |
| 最终定名 | OpenClaw | 确定品牌 |

**影响**：每次争议都助推了传播。

### 3.2 加入 OpenAI

**时间**：2026 年 2 月 15 日

Sam Altman 在 X 上宣布：
- Peter Steinberger 加入 OpenAI
- 负责"下一代个人智能体"开发
- OpenClaw 以开源基金会形式独立运营

**选择原因**：

| 竞争者 | 接触方式 | 优势 |
|--------|----------|------|
| Meta | 扎克伯格通过 WhatsApp 直接联系 | 社交生态 |
| OpenAI | CEO 亲自下场谈判 | 算力支持 + 愿景契合 |

Steinberger 的原话：

> "我完全可以把 OpenClaw 做成一家巨大的公司。但说实话，这对我来说并不兴奋。我的下一个使命是，做一个连我妈都能用的 Agent。"

## 四、技术架构

### 4.1 分层设计

```
交互层：WhatsApp、飞书、QQ、Telegram 等
    ↓
网关层（Gateway）：消息路由、会话管理、资源调度
    ↓
核心层：模型适配、技能系统
    ↓
技能市场：ClawHub
```

### 4.2 技能系统

**本质**：本地脚本包

**能力扩展**：

| 类别 | 示例 |
|------|------|
| 浏览器控制 | 网页抓取、自动化操作 |
| 代码操作 | 读写文件、执行命令 |
| 智能家居 | 控制设备、场景联动 |
| 文档处理 | PDF 分析、表格处理 |

## 六、生态竞争

### 6.1 竞品对比

| 项目 | 定位 | 目标用户 | 技术特点 | 健康度 |
|------|------|----------|----------|--------|
| OpenClaw | 企业级平台 | 技术团队 | Node.js，多 Provider | 健康 |
| ZeptoClaw | 个人管理 | 高级用户 | Rust 高性能 | 上升 |
| LobsterAI | 视频生成 | C 端用户 | Electron | 有隐患 |
| EasyClaw | 极简桌面 | 非技术用户 | 未披露 | 早期 |

### 6.2 关键分野

**平台 vs 产品**：

- OpenClaw/ZeptoClaw：开发者平台（配置驱动）
- LobsterAI/EasyClaw：终端产品（GUI 优先）

**性能哲学**：

- ZeptoClaw：Rust 追求极致性能
- OpenClaw：Node.js 功能完备性优先

## 七、安全挑战

### 7.1 ClawHavoc 事件

**时间**：2026 年 2 月

**发现**：

| 问题 | 数据 |
|------|------|
| 恶意技能数量 | 历史至少 1184 个 |
| 主要功能 | 恶意代码执行、反向 Shell、信息窃取 |
| 上传最多者 | ID hightower6eu，677 个恶意包 |

### 7.2 致命三角

同时具备三个条件时风险指数级放大：

1. 访问私有数据
2. 暴露于不可信环境
3. 能够对外通信

**OpenClaw 三个全中。**

**启示**：开放的技能生态带来能力，也引入安全风险。需要平衡好用与安全。

## 小结

OpenClaw 的爆发不是偶然。它击中了 AI 时代的核心需求：让 AI 从"回答问题"进化到"解决问题"。

4 个月登顶 GitHub，证明了一个道理：**真正的势能不在宣传声量里，而在每一次真实使用场景中自然涌出的需求。**

工具会过气，对工具本质的理解不会。
